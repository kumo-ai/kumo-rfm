# Contributing to KumoRFM

Thank you for your interest in contributing to KumoRFM! We welcome contributions that showcase the capabilities of KumoRFM through examples, notebooks, and demonstrations.

## How to Contribute

### 1. Fork and Submit a Pull Request

1. Fork the repository on GitHub
2. Create a new branch for your contribution
3. Make your changes
4. Submit a pull request to the main repository

### 2. Contribution Requirements

All contributions must meet the following requirements:

#### For Notebooks
- **All code cells must run successfully** - Test your notebook from start to finish before submitting
- **Must use KumoRFM** - Your notebook should demonstrate KumoRFM features and capabilities
- **Include clear documentation** - Add markdown cells explaining what your notebook does and how to use it

#### General Requirements
- Code must be nicely formatted and follow Python best practices
- Text should be clear, well-written, and easy to understand
- The example should be understandable end-to-end

### 3. Contribution Assessment Criteria

We evaluate contributions based on:

1. **Uses KumoRFM** - Does the contribution make proper use of KumoRFM?
2. **Novelty** - Is the contribution sufficiently different from existing examples?
3. **Code Quality** - Is the code nicely formatted and well-structured?
4. **Clarity** - Is the text easy to read and the end-to-end example understandable?

## What to Contribute

We're particularly interested in:

- **Notebooks** demonstrating new use cases or features
- **Applications** showing real-world KumoRFM usage
- **Benchmarks** comparing KumoRFM performance to alternative approaches

## Questions?

If you have questions about contributing, feel free to open an issue for discussion before starting work on a contribution.

Thank you for helping make KumoRFM better!
